Ninf Server 管理プロトコル

 

Ninf_callでは、登録されている関数の名前(entry nameとmodule name?)を用いて、呼び出すことができる。そのために、Ninf serverでは、そ の関数名とNinf Executable の対応表を保持している。

Ninf serverでは、Unix domainのlocal socket /tmp/ninf.registerを 用いて、管理プログラムとの通信を行って、Ninf Executable の登録を行っている。 そのプロトコルは、固定長のパケットを用いる。このプロトコルは ローカルにしか用いられないため、xdrを用いていない。

Ninf server 管理プロトコルには以下の種類のパケットがある。

  • NINF_REG_ENTER_STUB
    データ部に指定された Ninf Executable を登録する。具体的には、Ninf Executable を一度 起動し、インターフェイス を取得し、その名前とprogramを対応表に登録する。

  • NINF_REG_REMOVE_STUB (not implemented yet)
    データ部に指定された関数名を削除する。

  • NINF_REG_STOP (not implemented yet)
    Ninf Server を停止する。
