9.        Advanced Exercises

9.1.      Make the program fault-tolerant

ü        Ninf-G API detects faults (node down, network disconnection, etc.) and return appropriate errors to the application

ü        If you use tens to hundreds of servers and some of them will be down, you can simply discard the server, i.e. the execution of the program will not be terminated by a single fault.

 

9.1.1.      Exercise

ü        Prepare a server program that sleeps 300 seconds, then return hostname

ü        Write an IDL file

ü        Prepare a client program that calls the server

ü        Should check return code of each Ninf-G API

ü        Implement error handling code in the Client

ü        Run and kill server processes, then see what happens.
